     1# libmatch
     2
     3![build status](https://github.com/abhchand/libmatch/actions/workflows/test.yml/badge.svg?branch=main)
     4
     5A utility library for solving matching problems.
     6
     7`libmatch` supports solving the following classes of problems:
     8
     9| Matching Problem | Shorthand | Description |
    10|---|---|---|
    11| [Stable Marriage Problem](https://en.wikipedia.org/wiki/Stable_marriage_problem) | `SMP` | Matching between two groups of members |
    12| [Stable Roommates Problem](https://en.wikipedia.org/wiki/Stable_roommates_problem) | `SRP` | Matching within a group of members |
    13
    14`libmatch` can be used as a **standalone executable** or included **as a library** in your Go program.

    27## <a name="what-does-this-do">What Does This Do?
    28
    29Given one or more sets of *member preferences*, `libmatch` finds an optimal matching between members.
    30
    31<div align="center">
    32  <img src="https://github.com/abhchand/libmatch/raw/main/meta/matching.png" width="400px" />
    33</div>
    34
    35The result will be mathematically "stable", meaning no two members will prefer each other over their existing matches.
    36
    37`libmatch` provides solutions to solve variations of this classic matching problem, which has a wide range of practical applications.
    38
    39## <a name="as-an-executable">As an executable
    40
    41### <a name="installation"></a>Installation
    42
    43Download the the [latest release](https://github.com/abhchand/libmatch/releases/latest) for your platform.
    44
    45Or alternatively, build it from source:
    46
    47```shell
    48git clone git@github.com:abhchand/libmatch.git
    49cd libmatch/
    50
    51make all
    52```
    53
    54### <a name="usage">Usage
    55
    56Define member preferences as JSON data:
    57
    58```shell
    59$ cat <<EOF > input.json
    60[
    61  { "name":"A", "preferences": ["B", "D", "F", "C", "E"] },
    62  { "name":"B", "preferences": ["D", "E", "F", "A", "C"] },
    63  { "name":"C", "preferences": ["D", "E", "F", "A", "B"] },
    64  { "name":"D", "preferences": ["F", "C", "A", "E", "B"] },
    65  { "name":"E", "preferences": ["F", "C", "D", "B", "A"] },
    66  { "name":"F", "preferences": ["A", "B", "D", "C", "E"] }
    67]
    68EOF
    69```
    70
    71Run `libmatch`:
    72
    73```shell
    74$ libmatch solve --algorithm SRP --file input.json
    75A,F
    76B,E
    77C,D
    78D,C
    79E,B
    80F,A
    81```
    82
    83See `libmatch --help` for more options and examples
    84
    85## <a name="as-a-go-library">As a Go Library
    86
    87### <a name="installation"></a>Installation
    88
    89Add `libmatch` as a dependency:
    90
    91```shell
    92go get github.com/abhchand/libmatch
    93```
    94
    95### <a name="usage"></a>Usage
    96
    97Then `import` and use it:
    98
    99```go
   100package main
   101
   102import (
   103  "fmt"
   104  "os"
   105
   106  "github.com/abhchand/libmatch"
   107)
   108
   109func main() {
   110  prefTable := []libmatch.MatchPreference{
   111    {Name: "A", Preferences: []string{"B", "D", "C"}},
   112    {Name: "B", Preferences: []string{"D", "A", "C"}},
   113    {Name: "C", Preferences: []string{"D", "A", "B"}},
   114    {Name: "D", Preferences: []string{"C", "A", "B"}},
   115  }
   116
   117  // Call the solver for the type of matching algorithm you'd like to solve.
   118  // The top of the README contains a list of all supported shorthands.
   119  // In this case `SolveSRP` solves the Stable Roommate Problem
   120  result, err := libmatch.SolveSRP(&prefTable)
   121  if err != nil {
   122    fmt.Println(err)
   123    os.Exit(1)
   124  }
   125
   126  // libmatch returns a `libmatch.MatchResult` type above.
   127  // The mapping can be accessed via the `.Mapping` property
   128  for x, y := range result.Mapping {
   129    fmt.Printf("%v => %v\n", x, y)
   130  }
   131}
   132```
